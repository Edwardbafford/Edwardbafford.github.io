Mainly developed from the perspective of supervised learning.

Data

Validate the integrity of the training dataset.

Accuracy and Consistency

Errors in the data limits model performance and can be corrected or filtered out.

  • Input feature errors
  • Label errors
  • Inconsistency in the labeling

Inconsistent or unreliable data can be mapped to a new class for identifying uncertainty. At least the model will know it doesn't know!

Representative Training Data

Training data should contain input and output that are proportional to the population.

Models will optimize themselves to the training data and be sub-optimal when run on the true population.

Imbalanced Data

Outliers in a regression problem or a small portion of one or more classes in a classification problem.

Can cause performance issues on classes.

Solutions:

  • Upsampling
  • Downsampling (with bagging is an option)
  • Value minority data more while training model
  • Convert regression output to categorical pdf, show chances of an outlier

Model Output

First choosing what to model is important. Make sure what you are predicting is the best available target for the service you want to provide.

Then make sure the output type of the model is the best way to represent the target you're predicting. Obviously standard Regression and Classification outputs are an option.

Regression

Use a sigmoid function to restrict regression output within a range.

Multinomial Classification

Represent a range of values as a set of classes. The set of classes will represent a binned probability distribution function and can be more informative than a single regression value but will have less precision since each category represents a range.

Resize the number of classes to be more broad if necessary. Performance can be improved at the loss of precision.

Multilabel Classification

Tagging multiple categories per input can often be very useful.

Training multilabel classification models can be difficult. The dataset needs to have a examples of each class without too much correlation between any two classes. Thresholds need to be tuned and evaluated for each class.

A single multilabel classification model can be replaced with a collection of binary classification models, one for each category. This can increase performance by giving the models simpler problems but creates more complexity by having to train and manage multiple models.

Multitask Output

One model with multiple output types.

Only one model simplifies maintainence and multiple outputs can help generalize a powerful model preventing it from overfitting. It's not always an option depending on the model.

Model Types

Selecting the type of model to train usually depends on some combination of the following factors:

  • Implementation difficulty and maintainence
  • Interpretability
  • Training speed
  • Inference speed
  • Hosting resources needed
  • Problem complexity
  • Amount of training data

Bias, Variance, and Data

Complex problems need more powerful models to understand them. But more powerful models also need more data to counter the high variance (see bias-variance tradeoff) associated with powerful models.

Not enough data?

  • Break down the problem into a set of simpler problems
    • Cascade Ensemble
  • Apply boosting or bagging to high bias models
  • Apply regularization to a high variance model
  • Use domain knowledge to create high bias models with high performance
    • Generative Models
    • Bayesian Models

Boosting, Bagging, and Stacking

All can be used to help boost performance at the cost of creating and managing extra models.

Boosting

Cannot be easily integrated into a pipeline as an afterthought.

Model training needs to be does serially.

Inference can be done in parallel.

Often provides the best performance out of the other ensemble methods.

Bagging

Easy to implement or add to a pipeline.

Training can be done in parallel.

Inference can be done in parallel.

Models often do not cover each others weaknesses as well.

Stacking

Easy to implement or add to a pipeline.

Training and inference can only be done serially.
