Web Application Security Guide

I don't go into detail about any of the concepts here. Check my wiki for details about concepts mentioned here.


Software development process

Break down applications into smaller services to make security management easier.

Incorporate security into the code review process.

Use linters to discover vulnerabilities.

Continously test previously discovered vulnerabilities (regression testing).

Continously search for bugs using an internal team, bug bounty programs, or third party teams.

Prioritize security issues using a security ranking system, CVSS is a standard system.


Infrastructure

Use an encrypted protocol such as HTTPS to communication between machines. This prevents man in the middle attacks.

Don't trust be default

The principle is to setup the system in such a way that nothing is authorized by default and must be explicitly provisioned

  • Use whitelists over blacklists - as things change blacklists get outdated
  • Block ports by default - only the systems you are aware of can be reach externally
  • Only give systems access to the OS in ways they need - limits damage of a compromised system

Application Design

Passwords

  • Enforce high entropy passwords - harder to brute force hack
  • Save a hashed form of credentials, for authorization hash with the same algorithm and see if the hashed forms match - this prevents anyone from being able to steal user credentials
  • Offer multi-factor authentication

Front End

Ensure that user supplied data that is injected into the DOM and is interpretted as text. Linters can help with this.

API

GET requests shouldn't change the state of the application because GET requests can easily and accidentally be made by users.

Have the application check the origin of the requests and reject untrusted sources.

Use CSRF tokens to ensure the user has been using the application.

Back End

Disable external entity setting in XML parsers, or just use JSON! It's better anyway...

Use logging to time requests and high level processes to identify bottlenecks and DoS attacks.

Be careful when interfacing with other interfaces - OS, SQL, etc. For SQL always use prepared statements.

Regex

Check regex statements with regex linters to prevent poor performing regex.

Don't allow users to inject their own regex.
